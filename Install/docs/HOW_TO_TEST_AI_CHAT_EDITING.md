# როგორ გავტესტოთ ჩატიდან AI-ით კონტენტის ცვლილება

## რა გასწორდა

ჩატი არჩეულ ელემენტს აღარ უგზავნიდა backend-ს (interpret). ახლა **არჩეული ელემენტი** (სექცია + პარამეტრი) გადაეცემა interpret-ს, რათა AI ზუსტად იცოდეს რას უნდა შეცვლის.

---

## სად და როგორ ვტესტოთ

### 1. პროექტი

- გახსენი **ის პროექტი**, რომელსაც **უკვე აქვს Live Preview** (preview URL) და **მინიმუმ ერთი გვერდი** (მაგ. Home) **სექციებით**.
- თუ პროექტი ახალია ან ცარიელია, ჯერ **CMS / Builder**-ში შექმენი გვერდი და დაამატე სექციები (Hero, Features და ა.შ.), შემდეგ დააპუბლიშე ან შეინახე draft. მხოლოდ ამის შემდეგ იმუშავებს ჩატიდან შეცვლა.

### 2. ჩატის გვერდი + Inspect ჩანართი

1. **გახსენი პროექტის ჩატის გვერდი:**  
   `https://<თქვენი დომენი>/panel/projects/<project_id>` ან პროექტის არჩევის შემდეგ **Chat**.
2. **ჩანართებიდან აირჩიე "Inspect"** (არა "Chat" ვიზუალი და არა მხოლოდ ტექსტური ჩატი — უნდა იყოს **Inspect**, რომ მარჯვნივ ან ქვემოთ ჩანდეს **Live Preview**).
3. დაელოდე, სანამ **Live preview** სრულად ჩაიტვირთება (iframe-ში საიტის ხედი ჩანს).

### 3. ელემენტის არჩევა და ცვლილების მოთხოვნა

1. **Preview-ში დააკლიკე** იმ ტექსტზე/ელემენტზე, რომელიც გინდა შეცვლა (მაგ: Hero-ს სათაური, ღილაკის ტექსტი).
2. უნდა გამოჩნდეს **არჩევის ჰაილაითი** და ჩატის ინპუტთან **ტეგი** (მაგ: `HeroSection.headline` ან მსგავსი).
3. **ჩაწერე ჩატში**, მაგალითად:
   - „შეცვალე ეს სათაური: Welcome to Our Restaurant“
   - ან: „Change this title to Welcome“
4. **გაგზავნე** (Enter ან ღილაკი).

### 4. რას უნდა დაინახო

- **წარმატების შემთხვევაში:** ჩატში გამოჩნდება შეტყობინება მსგავსი „გაკეთებულია“ / „Applied“ და **Preview თავისით განახლდება** (iframe refresh) — შეცვლილი ტექსტი უნდა ჩანდეს.
- **შეცდომის შემთხვევაში:** ჩატში გამოჩნდება შეცდომის ტექსტი (მაგ: „Could not understand“ / „Failed to apply“). ამ დროს **Developer Tools → Console (F12)** გახსენი და ნახე, არის თუ არა წითელი შეცდომები ან failed request-ები (მაგ: `/ai-interpret-command` ან `/ai-site-editor/execute`).

## თუ არაფერი იცვლება — რას შევამოწმებთ

1. **Preview ჩანს?**  
   Inspect ჩანართზე უნდა ჩანდეს საიტის preview. თუ ჩანს „Loading“ ან ცარიელია, პროექტს შეიძლება არ აქვს preview URL ან გვერდები — დაბრუნდი CMS-ში და დაამთავრო გვერდის/სექციების დამატება.

2. **დაკლიკის შემდეგ ირჩევა ელემენტი?**  
   დააკლიკე ტექსტზე/სურათზე/ღილაკზე. უნდა გამოჩნდეს ჩარჩო/ჰაილაითი და ჩატის ინპუტთან არჩეული ელემენტის ტეგი. თუ არ ჩანს, შეამოწმე Console-ში JavaScript-ის შეცდომები.

3. **Backend მუშაობს?**  
   F12 → Network. გაგზავნე ბრძანება ჩატიდან. უნდა დაინახო:
   - `GET .../ai-site-editor/analyze` → 200, response-ში `pages` მასივი (მინიმუმ 1 გვერდი).
   - `POST .../ai-interpret-command` → 200, `success: true`, `change_set` with `operations`.
   - `POST .../ai-site-editor/execute` → 200, `success: true`.
   თუ რომელიმე 404/422/500-ს აბრუნებს, პრობლემა backend-ზე ან route-ებზეა.

4. **AI (OpenAI/Claude) კონფიგურირებულია?**  
   თუ interpret ან execute იძახება მაგრამ ყოველთვის „Could not understand“ ან ცარიელი change_set მოდის, შეამოწმე `.env`-ში AI API key და პროექტის/გლობალური AI კონფიგურაცია.
